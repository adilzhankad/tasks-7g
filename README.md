# Задача 1 — Анализатор энтропии сетевых пакетов

## Что это

Я написал консольную программу на Go, которая слушает сетевой интерфейс и анализирует трафик. Если коротко — она показывает, насколько "разнообразные" пакеты идут через сеть и с какими IP-адресами общается хост.

Что конкретно делает:
1. Подключается к сетевой карте и читает пакеты
2. Фильтрует — оставляет только пакеты, связанные с заданным IP
3. Считает энтропию по скользящему окну (последние N пакетов)
4. Каждые N секунд выводит результат: энтропию + список IP-адресов

## Как собрать

Сначала нужно установить Go и libpcap (если ещё не стоят):
```bash
sudo apt-get install -y golang-go libpcap-dev
```

Потом подтянуть зависимости и собрать:
```bash
go mod tidy
go build -o entropy-sniffer .
```

## Как запустить

```bash
sudo ./entropy-sniffer -i <интерфейс> -host <ip> [-window N] [-interval длительность]
```

`sudo` нужен потому что чтение сетевых пакетов — привилегированная операция, без root ядро не даст доступ к интерфейсу.

### Флаги

| Флаг | Что делает | По умолчанию |
|------|-----------|--------------|
| `-i` | Какой сетевой интерфейс слушать (например `eth0`, `enp4s0`) | обязательный |
| `-host` | По какому IP фильтровать пакеты | обязательный |
| `-window` | Сколько последних пакетов держать в окне | 100 |
| `-interval` | Как часто выводить результат | 5s |

### Пример запуска

```bash
sudo ./entropy-sniffer -i enp4s0 -host 192.168.10.3 -window 50 -interval 3s
```

### Что увидите в консоли

```

```

Тут видно: за последние 50 пакетов хост общался с тремя серверами, энтропия 3.25 — трафик разнообразный.

## Что такое энтропия

Энтропия Шеннона — это число, которое показывает, насколько разные пакеты летят по сети.

- **Энтропия ~0** — все пакеты одинакового размера. Скучный, однотипный трафик (например, только ping)
- **Энтропия 3–4+** — пакеты все разных размеров. Активный, разнообразный трафик (сайты, видео, DNS, всё вперемешку)

Формула: `H = -Σ p(x) * log2(p(x))`, где `p(x)` — доля пакетов размера `x` в окне.

## Почему я выбрал Go и libpcap

- **Go** — горутины позволяют легко ловить пакеты в одном потоке и выводить результат в другом. Плюс один бинарник на выходе, не нужно тащить зависимости
- **gopacket (libpcap)** — зрелая библиотека, поддерживает BPF-фильтры (фильтрация прямо в ядре — быстро и эффективно)
- **Не DPDK** — DPDK мощный, но требует hugepages, специальные драйверы, кучу настроек. Для этой задачи это перебор

## Как остановить

`Ctrl+C` — программа корректно завершится и закроет соединение с интерфейсом.
